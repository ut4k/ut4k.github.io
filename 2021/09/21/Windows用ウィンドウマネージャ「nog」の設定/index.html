Windows用ウィンドウマネージャ「nog」の設定

Windows向けに開発されているウィンドウマネージャ「nog」の設定について。
開発段階ということもありドキュメントにはあまり書かれていない部分があったためメモしておきます。

installation

githubからreleaseをダウンロード＞nog.exeを起動しようとしてもうまくいかない。
公式ドキュメントに従って、希望バージョンをcloneしたあと、
同梱の.download_release.ps1を利用するとよい。

成功すると自動的に下記の場所に実行ファイルを含め一式がインストールされる。

C:\Users\UserName\AppData\Roaming\nog

設定ファイル

C:\Users\UserName\AppData\Roaming\nog\config\init.lua

実行ファイル

C:\Users\UserName\AppData\Roaming\nog\bin\nog.exe

起動

これもドキュメントにある通り下記を実行。
※あとで簡易起動できるように設定しておいたほうがよさそう。

start-process $("$env:APPDATA\nog\bin\nog.exe")

以下はinit.luaに記載する。

バーの色を変更

nog.config.bar.color = 0x393837

値は先頭に0xをつけたRGBのHexで記載する。
注意すべきなのが、左からB->G->Rの順で記載しなくてはならないこと。
たとえばこの色だったら、
0x43b581と記載するとまったく意図せぬ色になってしまう。

正しくは0x81b543と記述する。
Numberを受け取るとあるが、クォーテーションで囲んだ文字列でもよさそう。

nog.config.bar.color = 0x81b543

Nerd Fontを表示

アイコンをそのままinit.luaのtextの値としてコピペすればよい。
(\\uE62B←こんな感じの記述は不要）
nf-custom-vim(vimのシンボル)を表示したいときは、
Nerd Fontのサイトからicon形式でコピペしてくればOK。

こんな感じでバーのコンポーネントにも使える。

local barStatusFancySymbol = function(symbol_)
  return {
    name = "FancySymbol",
    render = function()
      return {{ text = symbol_ }}
    end
  }
end

nog.config.bar.components = {
  left = {
		barStatusFancySymbol(" "),
  },
  center = {},
  right = {},
}

リサイズ機能をキーバインドする

ドキュメントのws_resizeの項に
ws_resize(direction)と1つのパラメータをとるような記述があるが、
詳細を読むと実際はws_resize(direction, amount)の２つをとるとのこと。

下記はようやく動かせた自分用設定。
対応キー押下でカレントウィンドウの幅を5ずつリサイズする。
キーは各アプリとの競合回避のため適当にCtrl-3、Ctrl-4のように設定。

nog.gbind("control+3", function()
-- nog.gbind("shift+h", function()
	nog.ws_resize("Left", 5)
end)

nog.gbind("control+shift+3", function()
-- nog.gbind("control+shift+h", function()
	nog.ws_resize("Left", -5)
	-- nog.ws_resize("Right", -5)
end)

nog.gbind("control+4", function()
-- nog.gbind("control+l", function()
	nog.ws_resize("Right", 5)
end)

nog.gbind("control+shift+4", function()
-- nog.gbind("control+shift+l", function()
	nog.ws_resize("Right", -5)
	-- nog.ws_resize("Left", -5)
end)

コンポーネントに任意の色を付ける

Barの項目を注意して読むと、
コンポーネントを構成するrenderファンクションが返すtableには、
じつはtextだけでなくfg、bgという、numberをとるフィールドも指定できるとのこと。
試してみると配色を変更できることを確認できた。

※色の指方法はnog.config.bar.colrの指定と同様。

A component text is a table with a required text field and optional fg, bg and value fields which take a number and change the colors of the component.

local foo = function(text_)
  return {
    name = "MyText",
    render = function()
      return {{
				text = text_,
				fg = 000000, 
				bg = 000000
			}}
    end
  }
end
